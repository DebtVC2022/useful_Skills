# 需求评审清单

## 1. 目标完整性

- [ ] 是否说明业务背景？
- [ ] 是否明确当前问题？
- [ ] 是否有可衡量成功指标？
- [ ] 是否说明不做该需求的影响？

## 2. 范围完整性

- [ ] 是否明确本期范围？
- [ ] 是否明确非范围？
- [ ] 是否有 MVP 切片？
- [ ] 是否定义范围变更触发条件？

## 3. 流程完整性

- [ ] 是否描述当前流程？
- [ ] 是否描述目标流程？
- [ ] 是否覆盖异常流程？
- [ ] 是否明确人工介入节点？

## 4. 数据完整性

- [ ] 是否列出所有核心字段？
- [ ] 是否说明字段来源？
- [ ] 是否说明字段类型、校验规则、默认值？
- [ ] 是否说明数据质量要求？

## 5. 原型完整性

- [ ] 是否有页面清单？
- [ ] 是否有页面层级结构？
- [ ] 是否有核心用户路径？
- [ ] 是否有页面状态：默认、加载、空、错误、权限不足？
- [ ] 是否有 Figma/MasterGo 可生成规格？

## 6. AI 场景完整性

- [ ] 是否明确 AI 任务类型？
- [ ] 是否说明输入数据？
- [ ] 是否定义输出结果？
- [ ] 是否说明准确率要求？
- [ ] 是否设计人工兜底？
- [ ] 是否设计数据飞轮？

## 7. 验收完整性

- [ ] 是否有可测试的验收标准？
- [ ] 是否覆盖主流程？
- [ ] 是否覆盖异常流程？
- [ ] 是否覆盖权限和数据边界？
