# DemandSpec 需求规范

## 1. 文档规范

- 所有缺失信息统一标记为 `待确认`。
- 所有推断信息统一标记为 `当前假设`，并说明风险。
- 不允许用“智能化、自动化、高效、便捷”等词替代明确功能和指标。

## 2. PRD 最低要求

PRD 至少包含：

- 背景与目标
- 范围与非范围
- 用户角色
- 主流程和异常流程
- 功能需求
- 页面与交互
- 字段与数据
- 业务规则
- 权限要求
- 验收标准
- 风险与依赖

## 3. AI 需求最低要求

AI 场景至少包含：

- AI 任务类型
- 输入数据来源
- 输出结果定义
- 准确率/召回率/采纳率等指标
- 人工兜底机制
- 错误样本回流机制
- 灰度或 POC 方案

## 4. 原型最低要求

原型规格至少包含：

- 页面清单
- Frame 结构
- 组件树
- 交互跳转表
- 状态设计
- 原型验收清单
